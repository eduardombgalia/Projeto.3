Este projeto tem como objetivo exercitar a programação de microcontroladores de 32 bits por meio do desenvolvimento de uma aplicação prática utilizando a plataforma ESP32 DevKit. 

Na Parte 1 do projeto, o foco principal está na configuração e controle básico de temporização e modulação de largura de pulso (PWM) utilizando os recursos já presentes no microcontrolador. O trecho mais crítico do código envolve o uso das funções clássicas da API do Arduino para ESP32, especificamente onde se determina a frequência e a resolução do canal PWM por meio de comandos estruturados de controle periférico. Ao iniciar o sistema, o código faz o mapeamento dos pinos de saída e define o ciclo de trabalho que define a potência média entregue ao atuador. O conceito central aqui é a modulação por largura de pulso, uma técnica onde a razão entre o tempo em que o sinal permanece em nível lógico alto e o período total da onda determina a tensão média simulada. Isso permite controlar a intensidade de componentes analógicos usando um sinal puramente digital. Os resultados dessa primeira etapa demonstram uma resposta linear e estável do componente controlado, validando que a temporização interna da ESP32 consegue manter a onda quadrada na frequência estipulada sem causar oscilações ou travamentos no fluxo principal do programa.


No Exercício 1 da Parte 2, temos a integração do servomotor. O trecho de código mais relevante nesta etapa destaca-se pela inclusão da biblioteca e pelas funções de inicialização onde o servo é vinculado a um pino digital específico, além do comando de escrita angular que traduz diretamente valores de graus para o sinal de controle. Para viabilizar essa aplicação, é empregada essencialmente a biblioteca ESP32Servo. Esta biblioteca é fundamental porque os temporizadores padrão do Arduino foram desenhados para microcontroladores AVR (como o Arduino Uno) e não funcionam corretamente na arquitetura de 32 bits da ESP32. A biblioteca ESP32Servo atua gerenciando diretamente os canais de hardware PWM internos do chip da Espressif, garantindo a geração exata de uma onda quadrada com frequência fixa de 50 Hz, o que é o padrão universal exigido para o correto funcionamento de servomotores.

 A discussão dos resultados nesta etapa demonstra que, ao utilizar a biblioteca dedicada, a ESP32 consegue converter variáveis numéricas simples de ângulos em temporizações de microssegundos extremamente precisas. A simulação aponta um movimento fluido e controlado do servo para as posições estipuladas no código, provando que o gerenciamento dos temporizadores internos feito pela biblioteca evita oscilações mecânicas indesejadas e garante a estabilidade do posicionamento angular do atuador em tempo real.

Por último, no Exercício 2 da parte 2, foi desenvolvido um sistema de controle PWM utilizando a plataforma ESP32 DevKit e a biblioteca nativa MCPWM. A aplicação proposta simula o controle de um atuador, em que o potenciômetro é utilizado para ajustar manualmente o duty cycle do sinal PWM gerado no GPIO 25. O valor analógico do potenciômetro é lido pelo conversor ADC da ESP32 e convertido para uma faixa de 0 a 100%, representando a variação de velocidade ou potência aplicada ao atuador. Além do controle do duty cycle, o sistema possui um botão utilizado para inverter o sentido de operação. A cada acionamento do botão, o programa alterna entre os estados horário e anti-horário. Para facilitar a visualização na simulação do Wokwi, dois LEDs foram utilizados como indicadores de estado: o LED verde representa o sentido horário e o LED vermelho representa o sentido anti-horário. As informações do sistema são enviadas continuamente pela comunicação serial, configurada em 115200 baud, permitindo o monitoramento dos valores de ADC, duty cycle e sentido de operação. Esses mesmos dados também são apresentados em um display OLED conectado à ESP32 pelo barramento I2C, permitindo visualizar diretamente os parâmetros de funcionamento durante a simulação. Dessa forma, o projeto demonstra a aplicação prática da biblioteca MCPWM em um sistema embarcado, integrando geração de PWM, leitura analógica por ADC, entrada digital por botão, GPIOs, comunicação serial e display OLED. A validação do duty cycle foi realizada por meio dos valores exibidos no Serial Monitor e no display OLED, enquanto os LEDs foram utilizados para representar visualmente o sentido de operação do sistema.
